写一个脚本，扫描所有 kotlin 测试，找到所有 backtick naming 的函数，写入一个文件。

读取这个文件，为每个 backtick naming 的函数给出对应的 camelCase 命名。

然后对每个 backtick naming 的函数，修改函数名为 camelCase 命名，并添加一个注解，说明原来的 backtick naming 函数名。

例：

```kotlin
// 原来的 backtick naming 函数
@Test
fun `should return true when input is valid`() {
    // 测试代码
}
```

```kotlin
// 修改后的函数
@Test
@DisplayName("should return true when input is valid")
fun shouldReturnTrueWhenInputIsValid() {
    // 测试代码
}
```
