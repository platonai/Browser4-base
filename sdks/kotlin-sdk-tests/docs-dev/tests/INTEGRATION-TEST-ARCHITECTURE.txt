                              Test Execution Flow
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                           │
│  1. Maven/Surefire starts JUnit 5 tests (default: groups=IntegrationTest) │
│                    ↓                                                     │
│  2. @SpringBootTest starts PulsarRestServerApplication                    │
│                    ↓                                                     │
│  3. Server starts on random port (e.g., 52341)                            │
│                    ↓                                                     │
│  4. TestServerConfiguration starts Mock EC Server (port 18080)            │
│                    ↓                                                     │
│  5. KotlinSdkIntegrationTestBase creates PulsarClient(baseUrl=localhost)  │
│                    ↓                                                     │
│  6. Test methods execute SDK operations via real HTTP calls               │
│                    ↓                                                     │
│  7. After each test: cleanup session and close client                     │
│                    ↓                                                     │
│  8. After all tests: Spring context shuts down                            │
│                                                                           │

                          Test Tag Strategy
┌─────────────────────────────────────────────────────────────────────────┐
│                                                                           │
│  Base class tags (in KotlinSdkIntegrationTestBase):                        │
│    @Tag("IntegrationTest"), @Tag("RequiresServer")                         │
│                                                                           │
│  Conventional tags used by tests:                                          │
│    @Tag("RequiresBrowser")    ← Needs Chrome/Chromium                      │
│    @Tag("RequiresAI")         ← Needs AI/LLM config (optional)             │
│    @Tag("Fast")               ← < 5 seconds                                │
│    @Tag("Slow")               ← > 10 seconds                               │
│    @Tag("ManualOnly")  ← Excluded by default, run manually          │
│                                                                           │
│  Default execution behavior (sdks/kotlin-sdk-tests/pom.xml):               │
│    - surefire.groups = IntegrationTest                                     │
│    - surefire.excludedGroups = ManualOnly,PassedOn20260203          │
│                                                                           │
│  Test Execution (Windows PowerShell, recommended from repo root):          │
│    - Run integration tests (default tags):                                 │
│        .\mvnw.cmd -pl sdks/kotlin-sdk-tests -am test                        │
│                                                                           │
│    - Run a single test class:                                              │
│        .\mvnw.cmd -pl sdks/kotlin-sdk-tests -am test -Dtest=WebDriverIntegrationTest
│                                                                           │
│    - Override tags (examples):                                             │
│        .\mvnw.cmd -pl sdks/kotlin-sdk-tests -am test -Dgroups="IntegrationTest,Fast"
│        .\mvnw.cmd -pl sdks/kotlin-sdk-tests -am test -Dgroups="IntegrationTest" -DexcludedGroups="RequiresBrowser"
│        .\mvnw.cmd -pl sdks/kotlin-sdk-tests -am test -Dgroups="IntegrationTest,RequiresAI" -DexcludedGroups=""
│                                                                           │
│  Notes:                                                                    │
│    - Do NOT rely on legacy flags like -DrunITs / -DrunSDKTests;             │
│      current behavior is driven by Surefire groups/excludedGroups.         │
│                                                                           │
